Design Document

By Yixiu Liu, Paul Nam, Nicholas Lombardi

INTRODUCTION

        

This is a design document for a stealth-based side-scroller called The Abyss. The game will employ tiled-based sprites, animations, physics engine, layered rendering, AI, and sound systems in the finished product.

TECHNOLOGY

        

The game, The Abyss, will be developed in the Phaser game framework, which is an open-source web game framework made by Photonstorm. Box2D will be used as the physics engine for collision detection and reaction. Artworks will be made in Gimp and photoshop, the all-purpose drawing tools, and GraphicsGale free edition, a sprite animation program.

BACKSTORY

        

Instead of waking up bed, the character woke up in a dream. Inhuman shadows roamed around the place. Weaponless and defenseless, the character must find a way to escape this nightmare without getting caught.

OBJECTIVE

        

The player is stuck in a dream. To escape, the player is to collect the necessary gate keys necessary to advance to the area. Since the player cannot put up a fight against the monsters, the player is to explore the map and obtain the keys while avoiding the monsters as much as possible.

GAMEPLAY

The game is a side-scroller. The character will have to use environmental factors or obtainable equipments to avoid the enemies and stealth to the next level of the map. Enemies will have fields of sight and hearing ranges. Player states (such as rate of heart beat/sanity and wounds) and actions (such as the different movement types, interaction with environment and usage of equipments) will affect the behavior of the AI.

Player can obtain equippable items that will assist them in stealth. Up to 2 can be deployed to each game level at once.

Player can interact with things that can affect the enemy alertness and behavior.

Different enemy types will have different behaviors and reactions.

CONTROLS

  • Up Arrow - Climb
  • Right/Left Arrow - Walk
  • Shift + Right/Left Arrow - Run
  • Down Arrow - Crawl
  • C - Jump
  • X - Interact with environment
  • A - Use equipment
  • Esc - Pause the game and bring up the game option menu

GRAPHICAL USER INTERFACE

  • Splash Screen
  • In-Game Menu
  • In-Game GUI

ARTWORK

        

Artworks will be original. Animations are required for the following:

  • Player Character
  • Idle
  • Walk
  • Run
  • Crawl
  • Jump
  • Interact
  • Death
  • Nightmare Monsters
  • Idle
  • Move
  • Alert
  • Attack
  • Other environment factors that affect the stealth

SOUND EFFECTS

        

Sound effects will be original. They will coincide with the following:

  • Moving on tile types
  • Crawling on tile types
  • Landing on tile types
  • Heart beat (insanity)
  • Interacting with environment
  • Monster groans

MUSIC

Music will be original, they will be added to best fit the atmosphere of the current game level.
